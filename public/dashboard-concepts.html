Clinic Portal · Dashboard Concepts

Three ways to give clinics
a home they actually open.

We explored three flavors of clinic-side dashboard for the pilot. Each is built around the data we have today — bookings, dentists, working hours, calendar mode — and gives the clinic owner a single place to see how the practice is doing and act on it.

A · Command Center
B · Analytics-First
C · Bento Grid · Recommended
Egypt's first integrated patient → dentist booking platform.
Built on Supabase + Next.js. Synced with Google Calendar.

Why we recommend it
  • · Best of A and B — both data and workflow on one screen
  • · Bento aesthetic = distinctly Dental Map, not generic SaaS
  • · Tiles are independent code — easy to add Reviews, Insurance, Retention later
  • · Scales gracefully from 1-dentist to 10-dentist clinics
The trade-off
  • · Slightly more design polish needed up front
  • · Some tiles look empty in cold-start phase (first 2 weeks)
Decision

At a glance.

Each direction emphasises different users. The bento grid wins because it serves both the owner *and* the receptionist on one screen, and the modular layout is the easiest to iterate on as we learn what pilot clinics actually look at.
