Dental Map · Clinic Dashboard · Design Brief v1 · Apr 28, 2026

Designing the clinic's daily home.

Brief for the designer working on the clinic-side dashboard. Three layout directions to choose from, real-world references for each, and the pick we'd build first if no one objects. The end goal is the page a clinic owner opens every morning to run their practice.

The project, in 5 lines
  • Product — Dental Map: Egypt's patient-facing dental booking marketplace, with live calendar sync into the clinic.
  • User — clinic owner (1–3 dentist clinics) checks the dashboard daily; receptionist uses it to confirm tomorrow's appointments and mark no-shows.
  • Data shown — bookings (today, week, month), revenue (sum of fees), filled-hours %, no-shows, dentist comparison, calendar sync status.
  • Stack — Next.js + Tailwind. Designer can use Figma; engineer will rebuild in Tailwind classes.
  • Locales — Arabic (default, RTL) and English. Layout must mirror cleanly.

Recommendation

Bento Grid (03).

It's the only one that survives our cold start: when a clinic has 5 bookings in their first week, a 30-day chart looks broken (kills 02), but a tile-based layout still feels alive. It gives the receptionist and the owner a reason to open it daily, where 01 is too ops-y for owners and 02 is too report-y for staff. And the modular tiles mean every future feature (reviews, insurance mix, retention) is one new tile — not a redesign.

Designer hand-off — must-haves
  • RTL parity — every layout mirrors cleanly for Arabic. Icons that imply direction (arrows, chevrons) flip; brand mark and KPI numbers don't.
  • Mobile — tiles stack into a single column. Today's schedule scrolls horizontally on mobile.
  • Empty states — every tile gets a designed empty state (e.g. "No bookings yet. Share your booking link →"). Don't leave them blank.
  • Skeletons — every tile loads independently; design a soft shimmer state per tile shape.
  • Receptionist mode (later) — same layout, but action queue is moved to top and KPI tiles are hidden. Plan for a role toggle.
  • Print/export — owner should be able to print a clean weekly summary. Assume a separate /report URL, not in-page.
